# Blender 守护进程性能测试指南

## 📊 验证守护进程是否运行

### 方法 1: 运行进程检查脚本

```bash
npx tsx check_blender_process.ts
```

输出示例（✅ 表示守护进程正在运行）：
```
🔍 检查 Blender 进程状态...

📋 Windows 进程列表检查:
✅ Blender.exe 进程正在运行！

📊 详细信息:
blender.exe              1234 Console   0     456,789 K
```

### 方法 2: 查看服务器状态接口

启动服务器后，在浏览器中访问：
```
http://localhost:3000/api/daemon-status
```

输出示例：
```json
{
  "daemonReady": true,
  "daemonExists": true,
  "daemonPid": 1234,
  "message": "Blender 守护进程正在运行"
}
```

### 方法 3: 查看服务器控制台日志

启动 `npm run dev` 后，应该看到：
```
[DAEMON] 正在启动 Blender 守护进程...
[Blender] [DAEMON] Blender 守护进程启动中...
[Blender] [DAEMON] 物体和修改器加载成功
[Blender] [DAEMON] 进入守护进程模式，等待参数请求...
[Blender] [READY]
[DAEMON] Blender 守护进程已就绪！
```

## ⚡ 性能测试方法

### 场景 A: 首次请求（第一次生成）

1. 启动 `npm run dev`
2. 等待看到 `[DAEMON] Blender 守护进程已就绪！`
3. 打开浏览器 `http://localhost:3000`
4. 观察右下角的耗时显示

**预期结果**：
- 首次请求耗时：2-4 秒（包括 Blender 启动时间）
- 浏览器控制台会看到 `✅ 生成完成，耗时: XXXX ms`

### 场景 B: 后续请求（参数改变）

1. 调整参数滑块（如"切片数量"）
2. 等待 100ms 防抖后自动生成
3. 观察右下角耗时和浏览器 Network 标签

**预期结果**：
- 后续请求耗时：**0.5-1.5 秒**（Blender 已加载，只需更新参数+导出）
- 这表示守护进程在正常工作

### 场景 C: 快速连续改变（测试守护进程的稳定性）

1. 快速拖动"切片数量"滑块（多次改变）
2. 观察每次请求的耗时
3. 检查是否有错误

**预期结果**：
- 每次请求都应该成功
- 耗时应该保持稳定在 0.5-1.5 秒
- 如果出现超时或错误，说明守护进程可能崩溃了

## 🔧 浏览器 DevTools 查看详细网络信息

1. 按 F12 打开开发者工具
2. 切换到 Network 标签
3. 调整参数，观察 `/api/generate` 请求

关键指标：
- **Time**: 总耗时（网络 + 服务器处理）
- **Response**: 返回的 modelUrl 和 duration
- **Console**: 生成完成的日志 `✅ 生成完成，耗时: XXXX ms`

## 🐛 故障排除

### 问题：耗时没有改进（每次都 3-5 秒）

可能原因：
1. ❌ Blender 守护进程没有正常启动
   - **解决**: 运行 `npx tsx check_blender_process.ts` 检查

2. ❌ 守护进程在上一个请求后崩溃了
   - **解决**: 查看服务器日志中是否有 `[ERROR]` 信息

3. ❌ .blend 文件太复杂，导致每次都需要长时间处理
   - **解决**: 简化模型，减少顶点数

### 问题：首次启动时出现错误

1. 查看服务器控制台是否有 `[ERROR]` 信息
2. 确认 Blender 路径正确
3. 确认 `.blend` 文件存在且格式正确

### 问题：参数改变后没有生成新模型

1. 检查浏览器控制台是否有 JavaScript 错误
2. 确保 `slice_count`, `length`, `thickness`, `twist_angle` 参数名拼写正确
3. 检查服务器日志中的 `[DAEMON] 收到请求` 是否有出现

## 💡 优化建议

如果守护进程已正常运行，但仍想进一步加速：

### 选项 1: 缩短防抖延迟
在 `src/App.tsx` 中修改：
```typescript
const timer = setTimeout(() => {
  handleGenerate();
}, 50);  // 改为 50ms（当前是 100ms）
```

### 选项 2: 优化 Blender 模型
- 减少 Geometry Nodes 的复杂性
- 使用 LOD（细节级别）系统
- 预烘焙尽可能多的计算

### 选项 3: 启用模型缓存
缓存相同参数的生成结果，相同参数秒级返回
